## 2. 系统测试
系统测试是确保软件系统可靠性和实用性的关键步骤之一。在进行系统测试之前，我们需要考虑可能会遇到的一些问题，并在测试过程中积极应对。

### 2.1 系统测试可能遇到的问题
1. 由于需求分析不清或开发过程中的误解，测试过程中可能会发现软件功能不完整、不符合需求或者存在逻辑错误的情况。
2. 系统在负载较大或者特定条件下的性能表现可能不佳，例如响应时间过长、资源占用过多等。这可能需要进一步优化系统的代码或者配置。
3. 软件可能存在安全漏洞，例如数据泄露、未经授权访问等，可能导致用户信息泄露或者系统被攻击。因此，需要进行安全性测试，以发现并修复潜在的安全隐患。

### 2.2 系统测试采用的方法
#### 2.2.1 单元测试
| 用户名 | 密码  | 预期结果 |
|------|------|------|
| 空值 | 11111 | 登录失败 |
| zs   | 空值  | 登录失败 |
| zz   | 11111 | 登录失败 |
| zs   | 11112 | 登录失败 |
| zs   | 11111 | 登录成功 |

#### 2.2.2 集成测试
集成测试的主要目的是验证不同模块之间的交互是否正常工作。在本系统中，由于各模块相对独立，模块间联系较少，因此单独测试各个模块后，整体系统一般不会存在太大问题。

#### 2.2.3 确认测试
将系统提供给教师和学生试用，测试系统各功能模块的功能是否能满足实际需求，并记录测试过程中发现的问题。
